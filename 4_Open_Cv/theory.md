# Задача 6 — робастный детектор движения

### 1. Видеопоток
Видео — это последовательность кадров.  
**Основная логика:**
*   открыть видео
*   читать кадры по одному
*   обрабатывать каждый кадр
*   показывать результат
*   останавливаться по клавише или концу видео

**В OpenCV используется:**
*   `cv2.VideoCapture()`
*   `cap.read()`
*   `cap.release()`

`cap.read()` возвращает:
*   `ret` — удалось ли прочитать кадр
*   `frame` — сам кадр

### 2. Проблема простого вычитания кадров
Простое сравнение двух соседних кадров реагирует на всё:
*   движение человека
*   тени
*   дождь/снег
*   листья
*   блики
*   шум камеры

Поэтому нужен более устойчивый метод.

### 3. Background Subtraction
**Идея:**
*   **фон** — то, что долго остаётся примерно неизменным
*   **передний план** — то, что появилось или движется

Для этого OpenCV использует модели фона, например:
`cv2.createBackgroundSubtractorMOG2()`

MOG2 строит статистическую модель фона и отделяет движущиеся объекты.  
**Важные параметры:**
*   `history` — сколько прошлых кадров учитывать
*   `varThreshold` — насколько сильным должно быть отличие от фона
*   `detectShadows` — учитывать ли тени

> Чем выше `varThreshold`, тем меньше ложных срабатываний, но можно пропустить слабое движение.

### 4. Маска движения
Результат background subtraction — бинарная или почти бинарная маска:
*   **чёрное** — фон
*   **белое** — движение
*   **серое** — тени, если `detectShadows=True`

С маской обычно делают дополнительную обработку.

### 5. Морфологическая очистка
Чтобы убрать мелкий шум:
*   **эрозия** удаляет маленькие белые точки
*   **дилатация** расширяет оставшиеся области
*   `opening` = erosion + dilation
*   `closing` = dilation + erosion

**Полезные функции:**
*   `cv2.erode()`
*   `cv2.dilate()`
*   `cv2.morphologyEx()`

**Ядро:**
*   `np.ones((3, 3), np.uint8)`
*   `np.ones((5, 5), np.uint8)`

### 6. Контуры
После получения чистой маски надо найти области движения:
`cv2.findContours()`

Контур — граница белой области на маске.  
Чтобы убрать мелкие ложные области:
`cv2.contourArea()`

**Принцип:**
*   если площадь контура слишком маленькая — игнорировать
*   если достаточно большая — считать движением

### 7. Bounding box
Для выделения объекта используется прямоугольник вокруг контура:
*   `cv2.boundingRect()`
*   `cv2.rectangle()`

### 8. Сигнал тревоги
Тревога — это логическое условие:
*   если найден хотя бы один крупный контур → движение обнаружено
*   иначе → движения нет

Можно вывести текст на кадр:
`cv2.putText()`

# Задача 7 — поиск логотипа через matchTemplate

### 1. Что такое template matching
Есть:
*   **большое изображение** — упаковка
*   **маленькое изображение** — логотип

Алгоритм двигает шаблон по большому изображению и считает, насколько хорошо он совпадает с каждым участком.  
**Функция:**
`cv2.matchTemplate()`  
Результат — карта совпадений.

### 2. Методы сравнения
Популярный метод:
`cv2.TM_CCOEFF_NORMED`

Он даёт значение похожести, обычно от плохого совпадения к хорошему.  
Для `TM_CCOEFF_NORMED` важно максимальное значение:
`cv2.minMaxLoc()`

**Тебе нужны:**
*   `max_val` — качество совпадения
*   `max_loc` — координаты лучшего совпадения

### 3. Порог совпадения
Логотип можно считать найденным, если:
`max_val >= threshold`

**Например, теоретически:**
*   0.8–0.95 — строгий порог
*   0.6–0.75 — мягкий порог

Точный порог зависит от качества изображения.

### 4. Координаты логотипа
`max_loc` — это левый верхний угол найденного логотипа.  
Чтобы получить правый нижний угол:
*   `x2 = x1 + ширина логотипа`
*   `y2 = y1 + высота логотипа`

Размеры берутся из `logo.shape`.

### 5. Проверка правильного положения
В задаче есть ожидаемое место логотипа.  
Теоретически нужно сравнить:
*   ожидаемые координаты
*   фактические координаты

Можно считать ошибку смещения:
`dx = abs(actual_x - expected_x)`  
`dy = abs(actual_y - expected_y)`

Если смещение больше допустимого порога — логотип нанесён неправильно.

### 6. Оптимизация через grayscale
Для ускорения можно перевести оба изображения в серый:
`cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)`

**Плюсы:**
*   меньше данных
*   быстрее вычисления
*   меньше влияния цвета

### 7. Масштаб и поворот
Обычный `matchTemplate` плохо работает, если логотип:
*   повёрнут
*   изменён по размеру
*   искажён перспективой

**Теоретические улучшения:**
*   искать логотип в нескольких масштабах
*   искать несколько повёрнутых версий шаблона
*   использовать ORB/SIFT/feature matching

Но для твоей задачи достаточно понимать, что базовый `matchTemplate` ищет почти такой же шаблон.

# Задача 8 — детектор углов Харриса и маршрут

### 1. Что такое угол в компьютерном зрении
Угол — это место, где яркость сильно меняется в двух направлениях.  
**Примеры:**
*   угол окна
*   стык стен
*   край крыши
*   пересечение линий
*   декоративный элемент фасада

### 2. Grayscale
Детектор Харриса работает с яркостью, поэтому изображение переводят в серый:
`cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)`

### 3. Детектор Харриса
**Функция:**
`cv2.cornerHarris()`

**Главные параметры:**
*   `blockSize` — размер области анализа
*   `ksize` — размер оператора Собеля
*   `k` — чувствительность детектора

**Типичные значения:**
*   `blockSize`: 2–5
*   `ksize`: 3
*   `k`: 0.04–0.06

Результат — карта откликов:
*   большое значение → вероятный угол
*   маленькое значение → не угол

### 4. Порог углов
Нужно оставить только сильные углы:
`corner_response > процент от максимального значения`

**Например:**
*   1% от максимума — много точек
*   5% от максимума — меньше, но надёжнее
*   10% — только самые сильные

### 5. Получение координат точек
После пороговой обработки надо получить координаты пикселей, где найдены углы.  
Теоретически это массив точек:
`(x1, y1), (x2, y2), (x3, y3)...`

**Важно помнить:**
В изображениях **y растёт вниз**.  
Поэтому:
*   нижние точки имеют большее y
*   верхние точки имеют меньшее y

### 6. Фильтрация точек
Харрис может найти слишком много точек рядом друг с другом.  
**Нужна фильтрация:**
*   убрать слишком близкие точки
*   оставить только сильные
*   оставить точки с большой площадью/яркостью отклика
*   ограничить количество точек

**Полезная идея:** если новая точка слишком близко к уже выбранной — пропустить.

### 7. Сортировка снизу вверх
Так как y растёт вниз:  
**снизу вверх = сортировка по y от большего к меньшему**  
То есть:
*   большой y → нижняя точка
*   маленький y → верхняя точка

### 8. Маршрут квадрокоптера
Маршрут — это линии между выбранными точками:
`cv2.line()`

Точки соединяются по порядку после сортировки.  
Для отметок точек можно использовать:
*   `cv2.drawMarker()`
*   `cv2.circle()`
*   `cv2.putText()`

### 9. Визуализация списка точек
Список угловых точек можно вывести в консоль:
`1: x=..., y=...`
`2: x=..., y=...`  
Или подписать точки на изображении.
